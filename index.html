一些想法、反思的记录

发表于 2020-12-13

2020年12月13日21:06:26
博客大致有了雏形。在搭建的过程中，按照教程一步一步来，有点心急地想要做出来一个可见的网站，于是就像个完全执行命令的机器人，没有关注、思考各步骤的作用和关系，没有形成一个认知框架。说是待搭建下一个生活方面的网站再细看。这样还是不太行。首先搭建过程中就会遇到问题，没有一个大概的理解会使解决问题的过程比较困难；其次现在又改变主意下一个网站要自己一行行代码完全自己写而不是用已有框架，原来的打算就没了，不过还是要返回去回顾理解的，保证如果要再一次搭自己能尽量不再看教程就做到。
在学习使用Markdown写第一篇文章的时候，每次先部署完再修改再部署花费了很多时间。但直到晚上快睡觉了才意识到一点，找到了可以可以在线预览Markdown语法写的文章的网页。简单步骤需要多次重复的时候不能憨憨浪费时间啊，希望下次遇到类似问题能早一点意识到。这也是新增这一篇文章的缘由吧。反思看看做的事情，从而寻找更优的解决方案。

Java错误：找不到或无法加载主类

发表于 2020-12-13 更新于 2021-01-06

问题

先javac编译产生class文件后
用java解释class文件时出现错误

原因

系统变量classpath设置问题导致无法找到class文件

解决方法

  1. classpath变量的路径最后加上;号则如未在所设置的路径找到class文件，会在当前目录下寻找
  2. 直接删除classpath变量

Scanner 使用

    next()只要遇到空格就结束了；
    nextLine()一次读一行\n

switch开关选择

语法

考点

  1. switch(表达式) ： 表达式的可用类型
     基本类型 ： byte short int char  (不可用： long boolean float double)
 引用类型 ： String enum枚举
  2. case 的常量值不能重复
  3. switch 具有穿透性
    case在执行过程中，只有遇到break才会停止并退出开关选择
    case在执行过程中,没有遇到break,则会穿透到它的下个case去继续执行，直到遇到break为止
    若case下方的所有case都没有break，则会逐个穿透（每个case都会执行一遍）
    除了业务需要手动穿透之外，以后所有的case都要写break

BUG

  1. java.lang.IllegalStateException: Scanner closed Scanner在使用前被关闭了
    解决方案： 在最后使用完成后再关闭Scanner
  2. java.lang.StringIndexOutOfBoundsException: String index out of range: 0 字符串索引越界(由break导致)
    解决方案 ： 重新创建新的Scanner
  3. java.util.NoSuchElementException 有很多Scanner,只要前面关闭一个，则接下来的其他Scaneer就找不到元素的
     解决方案： 所有创建的Scanner最后依次销毁
